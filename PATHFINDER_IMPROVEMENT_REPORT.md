## 4. 근본 원인 분석 (Root Cause Analysis)

### 4.1 문제 #1: AI 시드 생성의 구조적 결함

**현재 시드 생성 방식** (campaigns.json + AI 확장):
```
[시드] "청주 다이어트"
    ↓ AI 확장
[결과] "청주토요일오후다이어트한약가격후기추천"  ← 비현실적
```

**문제점**:
1. AI가 단어를 무작위로 조합하여 자연스럽지 않은 키워드 생성
2. 실제 사람들의 검색 패턴을 반영하지 않음
3. Naver 자동완성 데이터 활용 안 함

**증거**:
- "청주토요일오후매선침" - 이런 검색어를 치는 사람 없음
- 블로그 0개 = 수요 없음 = AI가 만든 가짜 키워드

### 4.2 문제 #2: KEI 공식의 맹점

**현재 KEI 공식**:
```python
KEI = (search_volume / doc_count) × adjustment_factor
```

**문제점**:
| 상황 | 검색량 | 공급 | KEI | 실제 가치 |
|------|--------|------|-----|-----------|
| 가짜 Golden | 10 | 0 | ∞ (100 처리) | ❌ 없음 |
| 진짜 Golden | 1,000 | 2,000 | 500 | ✅ 높음 |
| Red Ocean | 10 | 100,000 | 0 | ❌ 없음 |

**공급 0인 키워드가 높은 KEI를 받는 역설**:
- 블로그가 없다 = 수요가 없다 (아무도 쓰지 않음)
- 하지만 KEI 공식에서는 "경쟁 없음"으로 해석
- 결과: 가짜 Golden Keywords 대량 생성

### 4.3 문제 #3: 필터 전략의 딜레마

```
┌─────────────────────────────────────────────────────────────┐
│                    필터 강도 vs 결과                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  필터 엄격 (검색량 20+)                                      │
│  └─ 결과: 217개, KEI 500+ 1개                              │
│  └─ 문제: 좋은 longtail 놓침                                │
│                                                             │
│  필터 완화 (검색량 10+)                                      │
│  └─ 결과: 1,583개, KEI 500+ 1개                            │
│  └─ 문제: 가짜 Golden + Red Ocean 대량 포함                 │
│                                                             │
│  → 어느 쪽이든 KEI 500+ 개수 변화 없음!                      │
│  → 검색량 필터는 KEI 500+ 개수에 영향 미미                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**핵심 인사이트**:
- 검색량 필터만으로는 KEI 500+ 키워드를 늘릴 수 없음
- **공급(블로그 수) 필터가 없어서 Red Ocean이 걸러지지 않음**
- **공급 0인 가짜 Golden도 걸러지지 않음**

## 5. 개선 전략 (심층 재설계)

### 5.1 전략 A: 공급 필터 추가 (즉시 적용 가능)

**개념**: 공급량으로 가짜 Golden과 Red Ocean 제거

```python
# 새로운 필터 추가 (pathfinder.py)
MIN_SUPPLY = 10       # 블로그 10개 미만 = 수요 없음 (가짜 Golden 제거)
MAX_SUPPLY = 30000    # 블로그 3만개 초과 = Red Ocean (경쟁 불가)

if doc_count < MIN_SUPPLY:
    logger.debug(f"⚠️ Low supply (fake golden): {kw} (Supply: {doc_count})")
    continue

if doc_count > MAX_SUPPLY:
    logger.debug(f"🔴 Red ocean: {kw} (Supply: {doc_count})")
    continue
```

**예상 결과**:
| 필터 | 제거 대상 | 예상 제거 수 |
|------|-----------|--------------|
| Supply < 10 | 가짜 Golden | ~50개 |
| Supply > 30,000 | Red Ocean | ~800개 |
| **남는 키워드** | 양질 | **~700개** |

**장점**:
- 코드 수정 최소 (2줄 추가)
- 즉시 적용 가능
- 가짜 Golden과 Red Ocean 동시 제거

**단점**:
- 근본적인 시드 품질 문제는 해결 안 됨
- KEI 500+ 개수 증가 보장 안 됨

### 5.2 전략 B: 시드 생성 전면 재설계 (중기)

**개념**: AI 조합 대신 Naver 자동완성 기반 시드

```python
# 새로운 시드 생성 로직
def generate_real_seeds():
    base_seeds = ["청주 한의원", "청주 다이어트", "청주 교통사고"]
    real_seeds = []

    for seed in base_seeds:
        # Naver 자동완성 API 호출
        autocomplete = naver_autocomplete(seed)
        # 예: ["청주 한의원 추천", "청주 한의원 잘하는곳", "청주 한의원 비용"]
        real_seeds.extend(autocomplete)

    return real_seeds  # 실제 사람들이 검색하는 키워드!
```

**현재 vs 개선 비교**:
| 항목 | 현재 (AI 조합) | 개선 (자동완성) |
|------|----------------|-----------------|
| 시드 예시 | 청주토요일오후매선침 | 청주 매선 가격 |
| 자연스러움 | ❌ | ✅ |
| 실제 검색 여부 | 모름 | 확실히 검색됨 |
| 블로그 존재 | 0개 가능 | 반드시 있음 |

**장점**:
- 실제 검색되는 키워드만 수집
- 가짜 Golden 원천 차단
- 높은 품질 보장

**단점**:
- 구현 시간 필요 (Naver 자동완성 API 연동)
- 시드 다양성 감소 가능

### 5.3 전략 C: KEI 공식 개선 (중기)

**개념**: 공급 0인 경우 패널티 적용

```python
# 현재 KEI 공식
def calculate_kei(search_vol, doc_count):
    if doc_count == 0:
        return 100  # 문제: 가짜 Golden 생성
    return (search_vol / doc_count) * adjustment

# 개선된 KEI 공식
def calculate_kei_v2(search_vol, doc_count):
    # 공급이 너무 적으면 신뢰할 수 없음
    if doc_count < 10:
        return 0  # 가짜 Golden 차단

    # 공급이 너무 많으면 Red Ocean
    if doc_count > 50000:
        return 0  # Red Ocean 차단

    # 정상 범위에서만 KEI 계산
    return (search_vol / doc_count) * adjustment
```

**장점**:
- 가짜 Golden과 Red Ocean을 KEI 계산 단계에서 차단
- 필터 로직 단순화

**단점**:
- KEI 공식 변경은 다른 기능에 영향 줄 수 있음
- 기존 데이터와 일관성 문제
